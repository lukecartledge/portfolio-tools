# portfolio-tools — Agent Instructions

## Stack

- Node.js 20+, TypeScript, ESM (`"type": "module"`)
- Hono + @hono/node-server (local web server)
- Vanilla HTML/CSS/JS for review UI (no frameworks, no build step)
- chokidar@5 (folder watching, ESM-only)
- exifr (EXIF extraction)
- sharp (image resize for AI vision)
- @anthropic-ai/sdk (Claude Vision)
- contentful-management@12 (CMS publishing, plain client API)
- @clack/prompts (interactive CLI prompts)
- Vitest (testing)
- ESLint v10 + typescript-eslint (flat config) + eslint-config-prettier

## Conventions

- No `as any`, no `@ts-ignore`, no `@ts-expect-error`
- Strict TypeScript — all flags enabled
- ESM imports only (`import`, not `require`)
- Async/await over callbacks
- Early returns over deep nesting
- Named constants over magic numbers
- Error handling on all async operations
- No unused imports or variables
- `interface` over `type` for object shapes (ESLint `consistent-type-definitions`)
- `??` over `||` for nullish coalescing (ESLint `prefer-nullish-coalescing`)
- Tag format: lowercase-hyphenated (e.g. `new-zealand`)
- Atomic commits: each commit does one thing

## Project Structure

```

```

## Key Patterns

- Sidecar `.json` files live alongside each photo (same name, `.json` extension)
- Status flow: `pending` → `approved` → `published`
- Collections derived from subfolder name in watched directory
- All Contentful operations use Management API plain client (namespaced methods)
- Metadata merge: `userEdits` → `ai` → fallback (3-tier priority in `mergeMetadata()`)
- SEO fields: `seoTitle`/`seoDescription` generated by AI, editable in UI, published to Contentful
- Collection↔photo linking: `updateCollectionWithPhoto` sets `collection.photos` array (site reads this exclusively), `coverPhoto`, and `seoOgImage`
- Duplicate detection: checks sidecar `contentful.entryId` + slug-based Contentful lookup before publish
- Retry: `withRetry` wraps all external API calls with exponential backoff, jitter, and Retry-After support
- Collection auto-creation: `findCollection` → create if not found, cached within publish run

## Testing

- Vitest for unit tests (`npm test` or `npm run test:watch`)
- `tsc --noEmit` for type checking
- Co-located test files: `src/foo.test.ts` next to `src/foo.ts`
- Mock external SDKs (Anthropic, Contentful, sharp, exifr) — no real API calls in tests
- Use temp directories for filesystem I/O tests (sidecar read/write)
- CI runs Prettier → ESLint → tsc → Vitest on every PR
- `vi.hoisted()` for mock variables referenced inside `vi.mock()` factory functions
- Avoid `vi.resetModules()` + dynamic `import()` — causes vitest to hang on exit

## Do Not

- Add client-side JS frameworks (React, Vue, etc.)
- Add CSS frameworks (Tailwind, etc.)
- Add build tools for the frontend (Vite, webpack, etc.)
- Commit `.env` files
- Suppress type errors
